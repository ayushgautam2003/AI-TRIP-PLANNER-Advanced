# AI Trip Planner

A full-stack AI-powered travel itinerary generator. Users register, input trip preferences, and receive a complete day-by-day itinerary, hotel suggestions, and budget breakdown — all generated by GPT-4o Mini.

## Tech Stack

| Layer | Technology | Justification |
|---|---|---|
| Frontend | Next.js 15 + Tailwind CSS | App Router for clean routing; Tailwind for rapid, consistent UI |
| Backend | Node.js + Express | Lightweight REST API; easy JWT integration |
| Database | MongoDB + Mongoose | Flexible schema for AI-generated JSON itinerary data |
| AI | OpenAI GPT-4o Mini | Cost-efficient, fast structured JSON output via `response_format` |
| Auth | JWT (custom) | Stateless, simple, no third-party dependency |

## Setup (Local)

### Prerequisites
- Node.js 18+
- MongoDB running locally (`mongod`)
- OpenAI API key
- Google Places API key (for destination autocomplete)

### Backend

```bash
cd backend
cp .env.example .env
# Fill in OPENAI_API_KEY and JWT_SECRET in .env
npm install
npm run dev
# Runs on http://localhost:5000
```

### Frontend

```bash
cd frontend
# Edit .env.local — set NEXT_PUBLIC_GOOGLE_PLACES_API_KEY
npm install
npm run dev
# Runs on http://localhost:3000
```

## Authentication & Authorization

- Passwords hashed with **bcryptjs** (12 rounds) before storage
- On login/register, a **JWT** (7-day expiry) is issued and stored in `localStorage`
- Every protected API route passes through the `protect` middleware which verifies the JWT and attaches `req.user`
- Trip routes use `{ _id, userId: req.user._id }` in all DB queries — a user can never read or modify another user's trips

## AI Agent Design

A single OpenAI call per trip generation using `response_format: { type: "json_object" }` returns a structured object containing:
1. **itinerary** — array of days, each with activities (time, title, description, ticket price, rating)
2. **hotels** — 3 recommendations matching the budget tier
3. **estimatedBudget** — flights, accommodation, food, activities, total

The prompt is engineered to always return exactly `N` days, realistic prices for the destination, and budget-appropriate suggestions.

**Regenerate Day**: A focused prompt regenerates a single day's activities based on an optional user instruction (e.g. "more outdoor activities").

## Creative Feature: AI Packing List Generator

**What it does**: One click on the trip detail page generates a personalised packing list using the trip's destination, duration, budget type, traveler type, and actual planned activities as context.

**Why it solves a real problem**: Every traveller faces the same pre-trip anxiety — "Did I forget anything?" Generic packing lists don't account for whether you're backpacking vs. luxury travel, or whether your itinerary involves hiking vs. museum visits. This feature reads the AI-generated activities and produces a contextual checklist with:
- Categorised items (Documents, Clothing, Electronics, Health, etc.)
- Essential vs. optional flags
- Destination-specific extras
- A progress tracker to check off items as you pack
- Practical packing tips

**Engineering judgment**: Reuses the same OpenAI integration pattern (single structured JSON call), keeps state client-side (no DB storage needed for a checklist), and is lazily triggered — it doesn't run unless the user requests it, avoiding unnecessary API costs.

## Key Design Decisions & Trade-offs

| Decision | Trade-off |
|---|---|
| JWT in localStorage | Simple, but vulnerable to XSS. HttpOnly cookies would be more secure for production |
| Single OpenAI call for full trip | Fast and cheap, but limited context for very long trips (30 days) |
| MongoDB flexible schema | Easy to store AI JSON output, but no strict validation at DB level |
| No streaming | Simpler implementation; loading state shown instead of incremental display |
| Client-side auth state | Fast UX; risk of stale state if token is invalidated server-side |

## Known Limitations

- Google Places API key required for destination autocomplete
- OpenAI call may take 15–30 seconds for complex trips
- No email verification on registration
- Packing list is not persisted — regenerated on each visit
